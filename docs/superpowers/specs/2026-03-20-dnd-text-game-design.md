# 文字版龍與地下城跑團遊戲 — 設計文檔

## 概述

一款通過 Claude Code 終端交互的文字版 D&D 跑團遊戲。Claude 擔任地下城主（DM），玩家用自然語言輸入行動。遊戲狀態以 JSON/Markdown 文件持久化，支持多角色存檔、副本中途保存、NPC 隊友組隊。

## 核心設計決策

| 決策 | 選擇 | 理由 |
|------|------|------|
| 交互方式 | 純對話式 | 玩家已在 Claude Code 終端中，無需額外程序 |
| 規則複雜度 | 輕量化 D&D | 休閒向，重叙事輕計算 |
| 玩家模式 | 單人 + NPC 隊友 | 獨自遊玩也能挑戰高難度副本 |
| 語言 | 繁體中文 | 玩家偏好 |
| 存檔 | 多角色切換 | 不同角色可玩不同戰役 |

## 戰役世界

### 1. 艾澤拉斯（魔獸世界風）— 優先開發

- **風格**：經典中世紀奇幻，史詩感，陣營對立
- **種族**：人類、獸人、精靈（血精靈/暗夜精靈）、矮人、牛頭人、亡靈、巨魔、地精
- **職業**：戰士、法師、牧師、盜賊、獵人、聖騎士、薩滿、術士、德魯伊
- **特色機制**：陣營聲望、副本掉落裝備、天賦樹（簡化版）
- **副本設計**（按等級遞進）：
  - **初級（Lv1-10）**：死亡礦井、哀嚎洞穴
  - **中級（Lv10-20）**：影牙城堡、血色修道院、剃刀沼澤
  - **高級（Lv15-20）**：黑石深淵、通靈學院、斯坦索姆
  - **終局團本（Lv20）**：熔火之心（拉格納羅斯）→ 祖爾格拉布（哈卡）→ 黑翼之巢（奈法利安）→ 安其拉神殿（克蘇恩）→ 納克薩瑪斯（克爾蘇加德，最終挑戰）
  - NAXX 完整還原四大區（蜘蛛區、瘟疫區、軍事區、構造區）+ 薩菲隆 + 克爾蘇加德

### 2. 迷霧深淵（克蘇魯神話風）

- **風格**：暗黑恐怖、調查推理、理智崩潰
- **種族**：人類為主（學者、偵探、水手、記者等背景）
- **職業**：調查員、神秘學者、醫生、退伍軍人、記者、私家偵探
- **特色機制**：理智值（SAN）、線索系統、恐懼判定
- **副本設計**：詭異莊園、深海遺跡、異界裂隙

### 3. 血月獵殺（血源詛咒風）

- **風格**：哥特維多利亞、獸化瘟疫、噩夢與夢境
- **種族**：獵人（人類基底，可逐漸獸化）
- **職業**：近戰獵人、火藥獵人、秘法獵人、教會獵人
- **特色機制**：變身系統、血瓶回復、內臟攻擊（處決）、洞察值
- **副本設計**：亞南城區、禁忌森林、噩夢邊境、上位者領域

## 核心規則系統

### 屬性（六大基礎）

| 屬性 | 縮寫 | 說明 |
|------|------|------|
| 力量 | STR | 近戰攻擊、負重 |
| 敏捷 | DEX | 遠程攻擊、閃避、先攻 |
| 體質 | CON | 生命值、耐力 |
| 智力 | INT | 法術攻擊、知識檢定 |
| 感知 | WIS | 察覺、洞察、治療（克蘇魯中影響理智值） |
| 魅力 | CHA | 社交、領導、部分魔法 |

- 屬性範圍：1-20（初始通過點數分配或擲骰生成）
- 屬性調整值 = (屬性 - 10) / 2（向下取整）

### 戰鬥系統

1. **先攻**：每個參戰者擲 1d20 + DEX 調整值，決定行動順序
2. **攻擊**：擲 1d20 + 攻擊調整值 ≥ 目標 AC 即命中
3. **傷害**：命中後按武器擲傷害骰
4. **暴擊**：自然骰 20 = 暴擊，傷害骰翻倍
5. **技能檢定**：擲 1d20 + 相關屬性調整值 ≥ 難度等級（DC）
6. **死亡**：HP 降至 0 進入瀕死，隊友可救治，3 回合未救則死亡

### 掷骰機制

- Claude 使用系統隨機數生成骰子結果
- 每次掷骰明確顯示：骰子類型、結果、加值、最終數值
- 格式範例：`🎲 攻擊檢定：d20(14) + 3 = 17 vs AC 15 → 命中！`

### 升級系統

- 經驗值（EXP）通過戰鬥和完成任務獲得
- 每級提升：HP 增加、屬性可能提升、解鎖新能力
- 等級上限：20
- 升級時顯示可選能力，由玩家選擇

### NPC 隊友系統

- **招募**：在城鎮/安全區域可招募 NPC 隊友，最多 3 名（含玩家共 4 人小隊）
- **AI 控制**：戰鬥中 NPC 自動行動，遵循職業邏輯（坦克拉仇恨、牧師優先治療、DPS 輸出）
- **玩家指令**：玩家可下達戰術指令（「牧師先治療我」「法師用 AOE」）
- **個性**：每個 NPC 有性格特徵和背景故事，會在探索中產生對話
- **成長**：NPC 與玩家同步升級

## 副本/冒險系統

### 結構

- **城鎮/據點**：安全區，可交易、招募、接任務、休息
- **野外探索**：隨機遭遇、隱藏寶箱、支線事件
- **副本**：多房間的線性/半開放區域，有小怪、陷阱、謎題、Boss

### 副本中途保存

副本進度完整記錄在存檔中：

```json
{
  "dungeon_state": {
    "dungeon_id": "deadmines",
    "current_room": "goblin_foundry",
    "cleared_rooms": ["entrance", "mine_shaft"],
    "defeated_enemies": ["overseer_rhahkzor"],
    "found_items": ["mining_helmet"],
    "party_status": {
      "player": { "hp": 45, "max_hp": 60, "resources": {} },
      "companions": [...]
    },
    "events": ["lever_pulled", "bridge_lowered"]
  }
}
```

- 玩家可隨時說「保存退出」，下次說「繼續」即可從中斷處恢復
- Boss 戰中也可保存（保存在 Boss 戰開始前的狀態）

## 遊玩流程

### 開始新遊戲
1. 玩家說「開始遊戲」或「新建角色」
2. Claude 讀取 `game.md`，展示三大戰役選單
3. 玩家選擇戰役 → Claude 讀取對應規則和世界觀
4. 進入角色創建（選種族 → 職業 → 分配屬性 → 命名 → 背景故事）
5. 存檔創建，進入遊戲世界

### 繼續遊戲
1. 玩家說「繼續遊戲」或「讀取 [角色名]」
2. Claude 讀取存檔 JSON + 對應戰役文件
3. 展示上次離開時的狀態摘要
4. 從中斷處繼續

### 遊戲循環
```
城鎮（交易/招募/接任務）
    ↓
野外探索（隨機遭遇/事件）
    ↓
副本（戰鬥/謎題/Boss）
    ↓
返回城鎮（交裝備/升級/推進劇情）
```

## 交互規範

### DM 叙事格式
- 場景描述用沉浸式第二人稱叙事
- 戰鬥數據用格式化區塊顯示
- NPC 對話用引號標記
- 重要選擇用編號列表呈現

### 範例交互


## 設計邊界

### 不做的事
- 不寫遊戲引擎或腳本程序
- 不做圖形界面
- 不追求 100% 還原 D&D 5e 或任何原作規則
- 不做聯機/多玩家支持

### 後續可擴展
- 新增更多戰役世界
- 加入成就系統
- 加入角色之間的共享倉庫
- 加入PvP競技場模式（玩家角色 vs 玩家角色存檔）
